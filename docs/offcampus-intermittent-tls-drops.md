# 학외 접속 시 API가 한 번씩 실패하는 이유 (SNU 경계 TLS 인스펙션)

> 상태: **원인 규명 완료(2026-06-17)**. 앱·백엔드 측 버그 아님 → 코드 수정으로 "고칠" 대상이
> 아니라, **클라이언트 재시도로 흡수**하거나 정보화본부에 인프라 튜닝을 요청할 사안.
> 작성 맥락: "학외에서 cse.snu.ac.kr API를 찌르면 한 번씩 실패한다"는 증상을 실측으로 추적.

## 1. 증상

- **학외(off-campus)** 에서 `https://cse.snu.ac.kr` 요청이 **~0.3% 확률로 실패**한다.
- 실패는 HTTP 에러(4xx/5xx)가 **아니다**. 응답을 받기 전에 **연결 자체**가 끊긴다.
- **즉시 재시도(새 연결)하면 바로 성공**한다. 그래서 "가끔 한 번, 새로고침하면 됨"으로 체감된다.
- **학내에서는 재현되지 않는다** → 학외/학내를 가르는 경로상의 무언가가 원인.

## 2. 실측 (KT 망 `61.98.119.137`에서, 2026-06-17)

| 테스트 | 결과 | 의미 |
|---|---|---|
| DNS | A 1개(`147.46.92.174`), AAAA 없음 | DNS 라운드로빈/IPv6 이슈 아님 |
| `/` 30회 | 30/30 200 | 정상시 안정 |
| TCP 핸드셰이크만 200회 (`nc -z`) | 0 실패 | 순수 L4 SYN 레이트 문제는 아님(이 표본에선) |
| TLS 핸드셰이크만 200회 (`openssl s_client`) | 0 실패 | 순수 TLS도 대부분 정상 |
| 동시 50개 burst | 50/50 200 | 명시적 rate-limit/WAF 차단 아님 |
| **full HTTPS 600회 + 300회** | **3 실패 (~0.3%)** | 표본을 키우니 드러남 |

### 2-1. 캡처된 두 실패 모드 (단계별 타이밍)

curl `-w`로 연결 단계(`connect`=SYN완료, `appconnect`=TLS완료, `starttransfer`=응답시작)를 찍어
**서로 다른 레이어**의 실패임을 확인:

| | 모드 A (try469) | 모드 B (try487) |
|---|---|---|
| 에러 | `(35) Connection reset by peer` | `(28) timed out 7002ms` |
| `connect` (SYN완료) | **0.013s ✅** | **0.000 ❌** |
| `appconnect` (TLS완료) | **0.000 ❌** | 0.000 |
| total | 0.021s (즉시 RST) | 7.002s (무응답) |
| 즉시 재시도(새 flow) | **200 ✅** | **200 ✅** |

- **모드 A — TLS ClientHello 단계에서 RST.** TCP는 13ms에 정상 연결됐는데(`connect` 성공)
  TLS는 시작도 못 하고(`appconnect=0`) 즉시 reset. **장비가 TCP는 통과시켜 ClientHello/SNI를
  본 뒤** 연결을 끊었다는 뜻 — 단순 L4 패킷필터는 ClientHello를 보지 않으므로, 이는 **TLS/SNI를
  들여다보는 인스펙션 장비**의 시그니처다.
- **모드 B — SYN silent drop.** TCP 첫 패킷이 응답 없이 사라짐(`connect=0`, 7s timeout).
  REJECT(RST 즉시)가 아니라 **DROP(조용히 버림)** → 방화벽 정책의 전형.

### 2-2. 경로상 위치 (traceroute)

```
12  147.46.254.187   ← SNU 망(147.46.x) 진입
13  147.47.20.58      ← SNU 내부
14~20  * * *          ← 이후 서버까지 전부 무응답(ICMP 차단)
```

hop 13 직후부터 ICMP를 막는 장비가 있다 = **SNU 캠퍼스 경계의 인스펙션 장비가 그 지점**.
학외→학내(north-south) 트래픽만 이 병목을 지나가고, 학내(east-west)는 안 지나가므로
"학외에서만 가끔"이 정확히 설명된다.

## 3. 원인: 경계 NGFW/IPS의 TLS 인스펙션 (의도된 차단 아님)

정상 사용자를 일부러 끊는 게 아니라, **stateful 딥 인스펙션을 라인레이트로 수행하는 장비가
순간적으로 자원 한계에 닿으면 큐잉 대신 흘려보내는(drop/reset) 구조적 부작용**이다.
"빠른 실패 + 클라 재시도"가 장비 입장에선 가장 싼 과부하 안전밸브다. 기여 메커니즘:

1. **TLS 인스펙션 용량 초과 → load shedding.**
   복호화-검사-재암호화는 CPU·메모리·동시 SSL 세션 수에 빡빡한 한계가 있다. 순간 임계치
   (초당 신규연결·동시세션·CPU)를 넘으면 새 핸드셰이크를 **리셋**해 부하를 턴다. → 모드 A.
2. **세션 테이블 경합 → SYN drop.**
   모든 flow를 상태 테이블에 기록하는데, 연결 churn이 높으면 세션 삽입이 실패(테이블 포화·
   버킷 한도·new-conn-per-sec 캡)하고 SYN을 조용히 버린다. → 모드 B.
3. **액티브-액티브 방화벽 클러스터 + 비대칭 라우팅.**
   엣지는 보통 방화벽 여러 대를 ECMP로 묶는다. flow 패킷이 멤버 간 해시로 흩어질 때 상태
   동기화가 순간 어긋나면, 핸드셰이크 중간 패킷을 받은 멤버에 그 세션이 없어 RST/drop.
   **새 연결은 다시 해싱돼 깨끗한 멤버에 안착** → "재시도하면 바로 됨"의 전형.
4. **IPS 스캔/플러드 휴리스틱.**
   출발지별 SYN 카운트가 임계 근처면 정상 버스트도 일부를 스캔 의심으로 떨군다.

> 즉 "왜 이런 짓을 하나"의 답: **한정된 자원으로 모든 north-south 트래픽을 깊게 검사하려다
> 순간 압력에서 흘리는 누수**. 정상 동작의 부작용이라 앱에서 없앨 수 없다.

## 4. 대응 (통제 가능한 순서)

1. **연결레벨 에러에만 재시도 (가장 실효적).**
   새 flow가 즉시 복구되므로, **idempotent GET**에 한해 `ECONNRESET`/`ETIMEDOUT`/TLS 에러일 때
   1~2회 즉시 재시도하면 체감 실패가 거의 0이 된다.
   - ⚠️ **HTTP 5xx에는 재시도 금지.** 그건 백엔드 의미(409 충돌 등)라 이 네트워크 실패와 별개다.
     "fetch network error(응답 못 받음)"와 "응답은 받았는데 5xx"를 구분해 **전자만** 재시도.
2. **연결 재사용 (keep-alive / HTTP-2 coalescing).**
   새 TLS 핸드셰이크 한 번 = 인스펙터에 주사위 한 번. 커넥션 풀로 핸드셰이크 횟수를 줄이면
   노출이 준다. 브라우저 fetch는 이미 재사용하지만, **외부에서 스크립트로 API를 찌르는
   클라이언트**라면 keep-alive agent를 반드시 켠다.
3. **근본 원인은 SNU 정보화본부 소관.**
   경계 NGFW/IPS가 정상 핸드셰이크를 일부 리셋/드롭하는 것 → 서버 IP(`147.46.92.174:443`)를
   TLS 인스펙션 예외/화이트리스트로 빼거나, 장비 용량·클러스터 상태동기화를 튜닝 요청.
   앱 코드로는 못 고친다.

## 5. 혼동하기 쉬운 별개 이슈 (이건 간헐 아님)

인증/mutation이 실패한다면 그건 *결정론적*이고 다른 원인이다 — JSESSIONID가 `Secure`라
cross-origin 요청에 안 실리고 세션 없으면 302 OAuth로 빠진다(`CLAUDE.md` §아키텍처 참고).
"조건이면 항상" 실패라, 본 문서의 "~0.3% 랜덤" 증상과는 구분해야 한다.

## 부록. 재현 방법

```bash
# 실패율 + 단계별 타이밍 (실패 순간만 출력)
for i in $(seq 1 600); do
  out=$(curl -sS -o /dev/null --max-time 7 \
    -w "code=%{http_code} connect=%{time_connect} appconnect=%{time_appconnect}" \
    "https://cse.snu.ac.kr/api/v2/notice?pageNum=1" 2>&1)
  echo "$out" | grep -q "code=200" || echo "try$i: $out"
done

# 경계장비 위치
traceroute -w 2 -q 1 -m 20 cse.snu.ac.kr
```
